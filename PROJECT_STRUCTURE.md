# Project Structure

This document provides a brief overview of the files and directories in this project.

## Root Directory

- **.env.example**: Example environment variables file with the configuration keys required to run the project locally.
- **.gitignore**: Specifies files and directories that should be ignored by Git.
- **.importlinter**: Defines architecture rules to enforce dependency boundaries between project layers.
- **.pre-commit-config.yaml**: Configures pre-commit hooks used during development.
- **.python-version**: Defines the Python version used in this project.
- **PROJECT_STRUCTURE.md**: This file. Documents the current project structure.
- **README.md**: Main project documentation with overview, endpoints, configuration, and local run instructions.
- **main.py**: Local application launcher that starts the Uvicorn server with reload enabled.
- **pyproject.toml**: Project metadata, dependencies, and tool configuration.
- **uv.lock**: Lockfile used to keep dependency versions consistent.

## Main Application Package

- **app/**init**.py**: Marks `app` as a Python package.
- **app/main.py**: Creates the FastAPI application instance and registers HTTP routers.

## Adapters Layer

- **app/adapters/**init**.py**: Marks the adapters layer as a Python package.
- **app/adapters/http/**init**.py**: Marks the HTTP adapter package.
- **app/adapters/http/routers/**init**.py**: Marks the routers package.
- **app/adapters/http/routers/system_router.py**: Defines operational HTTP endpoints such as `/`, `/health`, `/info`, and `/ping`.
- **app/adapters/http/schemas/**init**.py**: Marks the HTTP schemas package for future request and response models.

## Application Layer

- **app/application/**init**.py**: Marks the application layer as a Python package.
- **app/application/use_cases/**init**.py**: Marks the use cases package for future application flows.

## Domain Layer

- **app/domain/**init**.py**: Marks the domain layer as a Python package.
- **app/domain/entities/**init**.py**: Marks the domain entities package for future business entities.

## Infrastructure Layer

- **app/infra/**init**.py**: Marks the infrastructure layer as a Python package.
- **app/infra/db/**init**.py**: Marks the database infrastructure package.
- **app/infra/logging/**init**.py**: Marks the logging infrastructure package.
- **app/infra/settings/**init**.py**: Marks the settings infrastructure package.
- **app/infra/settings/settings.py**: Centralizes application settings loaded from environment variables.

## Development Directories

- **.git/**: Internal Git repository data.
- **.import_linter_cache/**: Cache generated by Import Linter.
- **.ruff_cache/**: Cache generated by Ruff.
- **.venv/**: Local Python virtual environment with installed dependencies.
