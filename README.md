Backend API for creating and managing internal support tickets, built as a portfolio project during the Backend with Python mentorship.

## Overview

The goal of this project is to build a realistic backend application to handle internal support requests such as IT demands, system access issues, operational problems, and internal service workflows.

## Project Scope

The system is expected to include features such as:

- user registration
- login
- ticket creation
- ticket listing and filtering
- comments on tickets
- ticket assignment
- status updates
- history tracking
- metrics exposure
- simple automations
- simulated AI-based classification suggestions

## Project Roadmap

The project will be built incrementally, moving through stages such as:

- initial in-memory implementation
- improved application structure
- database integration
- authentication
- tests and logging
- operation closer to a production-style project

## Planned Stack

- Python 3.11+
- FastAPI
- Uvicorn
- Pydantic
- PostgreSQL
- SQLAlchemy
- Alembic
- python-dotenv
- Docker
- GitHub Actions
